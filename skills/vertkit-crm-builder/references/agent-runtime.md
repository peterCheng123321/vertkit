# Agent Runtime Reference

Use this when adding agent workflows, approvals, enrichment, intake, automation, or human-in-the-loop flows.

## Architecture Rule

The TypeScript agent runtime is the orchestration layer, not the CRM backend.

- VertKit Go API owns CRM business records.
- Postgres owns durable CRM state.
- The TypeScript runtime owns provider calls, tool routing, permission checks, and audit events.
- Agent tools commit CRM changes by calling VertKit REST APIs with tenant context.

Never let agent memory, tool results, or audit events become the only source for an Account, Contact, Opportunity, Product, Rule, or Tenant.

## Good Agent Runtime Use Cases

- Lead intake that asks follow-up questions and creates Account/Contact/Opportunity records.
- Approval flow for large deals, discounts, or stage changes.
- Data enrichment that proposes updates and asks for review.
- Deduplication workflow that suggests merges.
- Background workflow that evaluates rules before committing a change.

## Required Integration Contract

Each agent job must carry:

- Tenant ID.
- Service credential.
- Actor/user context when available.
- Idempotency key for commit operations when retries are possible.
- Trace/request ID so Go API logs and agent audit events can be correlated.

Agent tools should:

- Fetch current state through VertKit APIs.
- Produce proposed changes.
- Run rules/evaluation APIs before commit when relevant.
- Commit through VertKit APIs.
- Store job and audit state in the agent service once persistence is added.

## Anti-Patterns

- Writing directly to VertKit Postgres tables from agent code.
- Using agent scratch state as the CRM record store.
- Letting LLM-generated JSON bypass domain/API validation.
- Running jobs without tenant context.
- Treating failed API writes as successful job completion.

## First Agent Runtime Milestone

The current first slice lives in `apps/agent`:

- Node/TypeScript HTTP endpoint for agent jobs.
- Tool registry with risk levels.
- Risk-based permission policy.
- OpenAI-compatible provider adapter.
- CRM API tools generated by hand from the stable VertKit API surface.
- Tests using Node's built-in test runner.
